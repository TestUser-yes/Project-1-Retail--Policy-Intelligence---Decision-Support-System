WHAT YOU HAVE:

  ✓ Orchestrator (intelligently routes queries to right agent)
  ✓ LangFuse Tracing (observes all agent executions)

================================================================================
                         HOW TO SHOW IT IN DEMO
================================================================================

OPTION 1: Quick API Test (2 minutes)
─────────────────────────────────────

1. Start server:
   $ python -m uvicorn app.main:app --reload

2. Get demo token:
   $ curl http://localhost:8000/api/token

   Copy the access_token value

3. Test RAG Agent (Policy Retrieval):
   $ curl -X POST http://localhost:8000/api/ask \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"query": "What is the data retention policy?"}'

   LOOK FOR: "agents_used": ["rag_agent"] in response

4. Test SQL Agent (Database Query):
   $ curl -X POST http://localhost:8000/api/ask \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"query": "How many vendors do we have?"}'

   LOOK FOR: "agents_used": ["sql_agent"] in response

5. Test Hybrid Mode (Both Agents):
   $ curl -X POST http://localhost:8000/api/ask \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"query": "Which vendors comply with our encryption policy?"}'

   LOOK FOR: "agents_used": ["rag_agent", "sql_agent"] in response

6. View agent execution details:
   In each response, look for "agent_details" array:
   {
     "agent_details": [
       {
         "agent_name": "RAG Agent",
         "status": "success",
         "latency_ms": 245.3,
         "confidence": 0.92,
         "data_source": "PDF Documents"
       }
     ]
   }

================================================================================
                        TROUBLESHOOTING
================================================================================

Q: agents_used is empty?
A: Check that orchestrator.py returns agents_used and agent_details

Q: agent_details not in response?
A: Verify api.py includes them in AskResponse model

Q: LangFuse not showing traces?
A: Set LANGFUSE_PUBLIC_KEY and LANGFUSE_SECRET_KEY in .env, restart server

Q: Query routing seems wrong?
A: Check _detect_intent() logic in orchestrator.py for keyword matching
